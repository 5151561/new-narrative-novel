Calm Review Desk

主舞台安静、文档感强，Proposal 栈是视觉中心。参考偏向 Outline / Wiki.js 的沉静阅读感。

适合：你想让 Scene 看起来更像“导演台 + 审稿台”，而不是 IDE。

Review Cockpit

更强调运行态与裁决节奏，右侧 Inspector 和底部 Dock 的存在感更高。参考偏向 Logseq 右栏 + BookStack tri-layout。

适合：你想让 Scene 明显带有 orchestration cockpit 的专业感。

C. Structured Studio
Scene Breadcrumb | View Title | Preset
Setup / Execution / Prose
Objective
Runtime Preset
Beat Rail
Proposal Grid / Stack
Accepted Summary + Next Actions

Structured Studio

结构更规整，Setup / Execution / Prose 的视图连续性更强。参考偏向 AppFlowy 的层级组织和 view title bar。

适合：你想让 Scene 的三段式切换更统一，减少“Execution 太像独立页面”的感觉。

我当前的建议

推荐先从 B. Review Cockpit 出发，再吸收 A 的安静阅读感，避免页面重新滑回“文档站”或“数据库页”。
